1. Introduction

2023 was a productive year for me. I had spent more time producing solutions with my current knowledge than I had obtaining more know-how. Nevertheless I could still find some time to explore new areas of SAP.

2. Projects

2.1. PPDS/OPTI

By far the biggest project was creating a PPDS optimizer heuristic for optimizing production and keeping together resources that are bound to the same sales order. This may not sound that difficult but I can assure anyone that the complexity of the problem on hand is in extreme heights. There are many-many smaller and bigger rules to find the right intersection of data and processing an entangled set of graphs poses its own challenges. It took more than half of 2023 full time to bring this solution alive and took time from the other half as well with ad-hoc refinements of the logic.

What this project has taught me is that it is worth using ABAPGit for tracking source code changes. I have been using git ever since I have started my SAP career, but not when doing ABAP. However, administering at around 60 ABAP classes along with their interfaces (long live OO) can benefit from using git instead of relying solely on the transportation system. Experimenting in git branches, reverting commits by a few commands is a lot easier than manually keeping order.

The other aspect of this project is that OO design patterns (hello GOF) are a must if you want to keep such a monster performant yet flexible. And not just that but ABAP Unit as well. Because no consultant and no client will be able to test the rules buried in such a multi-layered solution it is really nice to come prepared for those details and testing them before any real data touches them - or better yet before the productive code is written. Using test driven development (TDD) one will think about testability before the productive code is written and that makes for better planned classes and applications.

So, using TDD, ABAPGit and design patterns were a good decision retrospectively as well.

2.2. Workflow migration from ECC to S/4

The migration consisted of utilizing My Inbox instead of the old dedicated app that is no longer available on S/4. I am not a workflow developer, I know just enough as a Fiori/ABAP dev to migrate the existing workflow to My Inbox.

This was an ugly task, since the complexity of the workflow and the utilized 'unique' solutions in it made this a challenge. Although most of the transition is done there is still an open issue due to commit control changing in S/4 making a previous user exit unusable leading to dumps.

The takeaway: no one can judge the depth of a mess a workflow may hide in itself. As a result, no manday commitment can be made in such cases.

2.3. HR Fiori apps

Developed 4 Fiori apps, of which 3 were Fiori Elements and 1 freestyle. As a Fiori dev this was an easy task.

It was justified again that using Fiori Elements is a reasonable choice where feasible. However, tasks that seem simple and straight forward in freestyle may take a lot of time in Fiori Elements. Like creating a filter that accepts a date range but is filled with the current month as default, hiding all other options.

And creating wireframes and mockups was justified again.

2.4. My first RAP application

I have been eyeing an opportunity to create an application using the SAP Restful ABAP Programming Model and 2023 gave a chance for that.

The app itself was simple but it was a good requirement for a first application where everything had to work. I did not like that OData V4 only worked if the draft capability was enabled. Since the app was simple draft handling would have made it more complex that it would have helped and as such I have resorted to a V2 service binding.

I found the technology usable (managed scenario was available, since it was developed on a 2022 S/4) and I would be glad to work with it again as the separation of concerns guides developers and tries to free them from the function module-perform-thousand-line programs hell. I believe that when the extension part of the technology will be in order we will see more and more standard apps as well.
